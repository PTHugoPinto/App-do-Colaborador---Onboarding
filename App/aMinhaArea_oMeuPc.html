Deverá abrir um Incident sempre que um equipamento, programa ou serviço parou ou deixou de funcionar como estava anteriormente.

Deverá abrir um Service Request sempre que precisar de uma mudança ou necessitar de algo adicional ao que tem atualmente.

As categorias que irá encontrar para o Service Request:

  • Exchange – Change Mailbox : Exchange – Change Mailbox - Adjust SMTP alias, modify settings of booking room.
  • Local IT support
  • Hardware Request
  • S/MIME Personal Certificate Request/Recovery (email Encryption)
  • ADAM Tool Services - Advanced change of account properties : i.e. Change login/user's name, extend domfsc01 admin account's expiration date, other modification in AD
  • File Service – Modify Polish File Share : Permissions adjustment for shared folders located on: \\g02.fujitsu.local\dfs\LDZT and LDZU locations.
  • ADAM Tool Services - Simple change of account properties : i.e Change display name, extend G02 account's expiration date
  • Firewall Rule Change : Used to order, change or delete Firewall Rules
  • EMEIA Workstation Exception Request : Exception for VM installation or any other related to workplace service

Para abrir Ticket/Incident aceda ao Portal Ask IT e na página selecione New Incident .

Ao abrir um novo Ticket/Incident irá preencher um pequeno formulário. Estes são os campos que serão de preenchimento obrigatório:

  • Affected end user
  • Your Telephone
  • Email
  • Type of incident (DropDown menu)
  • Short Description
  • Description

Os direitos administrativos dos postos de trabalho para todas as contas foram removidos. Quando os direitos administrativos forem exigidos, irá aparecer o popup UAC, mas o nome de utilizador e a palavra-passe não permitirão direitos de administração. Em vez disso, os utilizadores terão de solicitar uma conta de administração temporária para a sua estação de trabalho a partir do Portal de Gestão de Conta de Diretório Ativo (ADAM).

Haverá 3 tipos de conta de administração e os utilizadores poderão candidatar-se à conta mais aplicável à tarefa/atividade que precisam de realizar.

  • Aprovação temporária de conta de administração de 3 dias – auto-aprovação (normalmente dentro de 30 minutos).
  • Aprovação temporária de conta de administração de 4 semanas – requer aprovação do Manager.
  • Aprovação permanente de conta de administração de 1 ano – requer aprovação do Manager e reconfirmação anual das necessidades empresariais*

Para a maioria dos utilizadores, a conta temporária de 3 dias auto-aprovada será suficiente, mas alguns grupos de utilizadores podem exigir direitos de administração numa base mais regular.


Poderei utilizar as minhas credenciais de utilizador para interromper tarefas no Task Manager?

  • Ainda poderá operar o Task Manager, mas apenas para as tarefas que definiu; As tarefas do sistema precisam de acesso administrativo para interagir com o que não será capaz de parar estas tarefas.

Ainda poderei instalar aplicações?

  • Ainda poderá descarregar e instalar aplicações a partir das aplicações/software fujitsu, uma vez que estas são entregues via SCCM. Você NÃO será capaz de descarregar software aleatório de outras fontes.

Ainda poderei usar as minhas smart printer apps sem direitos de administração?

  • Não será possível utilizar Apps como a HP Smart com uma conta padrão.
